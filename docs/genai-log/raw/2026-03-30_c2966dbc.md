> ### 7. KEY DIVERGENCES FROM Be NAMING
> 
> | Pattern | Be Approach | Pane Approach | Rationale |
> |---------|------------|--------------|-----------|
> | **Event Dispatch** | Runtime `BMessage::what` integer matching | Exhaustive enum pattern matching | Type safety, compiler verification |
> | **Handler Chain** | `BHandler::SetNextHandler()` delegation | Single `Handler` per pane + `fallback()` trait method | Rust's trait system replaces C++ single-dispatch workaround |
> | **Observer Pattern** | `BHandler::StartWatching()` pub/sub | Filesystem attributes (`/pane/{id}/attr/*`) + `pane-notify` watcher | Persistence, crash recovery, scriptability |
> | **Timers** | Separate `BMessageRunner` service | Integrated into `Messenger` (`send_delayed`, `send_periodic`) | Simpler API; `recv_timeout` makes looper-local timers natural |
> | **Visibility** | Refcounted `Hide()`/`Show()` | Boolean `set_hidden()` (idempotent) | Compositor owns visibility; no refcounting needed |
> | **Return Convention** | Side-effect based (e.g., `PostMessage(B_QUIT_REQUESTED)`) | Return value controls loop (`Ok(false)` = exit) | Rust Result type; clearer control flow |
> | **Messenger Scope** | Single direction (target) | Dual direction (compositor + self-delivery) combined | One handle for all messaging from a pane |
> | **Request-Reply** | `SendReply()` on BMessage + optional handler target | Session-typed `ReplyPort` + message enum variants | Typed, exhaustive, prevents deadlock |
> | **Monitoring** | (no Be mechanism) | `Messenger::monitor()` + `Message::PaneExited` | Erlang-style crash propagation |

> ### Summary
> 
> Pane maintains **strong semantic correspondence** to BeOS concepts (App ŌåÆ BApplication, Pane ŌåÆ BWindow, Handler ŌåÆ BHandler, Message ŌåÆ BMessage) while **systematically adopting Rust idioms**:
> 
> 1. **Types are named directly** (BApplication ŌåÆ App, not ApplicationKit)
> 2. **Traits replace virtual classes** (Handler is a trait with default methods, not an abstract base)
> 3. **Enums replace untyped dispatch** (Message enum vs. BMessage's `what` code)
> 4. **Return values replace side-effects** (Ok(false) exits vs. PostMessage(B_QUIT_REQUESTED))
> 5. **Builders replace untyped setters** (Tag builder vs. BWindow constructor chaos)
> 6. **Filesystem replaces pub/sub** (attributes watched by external tools vs. built-in StartWatching)
> 7. **Session types replace protocols** (typed handshake vs. capability code matching)
> 
> The result is a modernized Application Kit that preserves Be's distributed actor model while leveraging Rust's type system to prevent entire categories of concurrency bugs.</result>
