## Key decisions this session

1. **RequestProtocol supertrait** (option B) — three-round four-agent deliberation. Separate trait, not associated type default. send_request requires P: RequestProtocol. HandlesRequest<P> trait for request handlers.
2. **Unified ReplyPort** — closure-erased backend (`Box<dyn FnOnce(Result<T, ReplyFailed>)>`). One type for in-process and wire replies. T: Serialize bound on wire constructor only.
3. **Wire format hardening** — u8→u16 session discriminant (65534 sessions, monotonic, reconnect-on-exhaustion for Phase 2). Protocol tag (XOR-fold of ServiceId UUID) prepended to service payloads, checked before deserialization.
4. **Codec self-poison** (S3) — AtomicBool, all reads after any FrameError return Poisoned.
5. **Dispatch entry orphaning fix** (S1) — fail_session method, server echoes ServiceTeardown to revoker, LooperCore handles ServiceTeardown.
6. **Bounded write channel** — sync_channel(128) for backpressure. Per-pane threading isolates stalls (same as Be's per-window model). ServiceHandle::Drop uses try_send.
7. **dispatch_service catch_unwind** — bug found by stress tests, fixed.
